🔧 Root Cause Analysis

The CBOE cached data in data/cboe_complete/spx/0dte/ was previously downloaded (based on timestamp: May 25, 2026). However, the data structure suggests issues:

Sample row from 2023-02.csv.csv.gz:
ticker,expiration,strike,option_type,timestamp,open,high,low,close,bid_open,bid_high,bid_low,bid_close,ask_open,ask_high,ask_low,ask_close,delta,gamma,theta,vega,rho,iv,volume,open_interest,underlying_price

Observed:
- underlying_price = 0.0 (always)
- bid_close / ask_close = 0.0 (most rows)
- close = nonsensical large values

Likely Issue: The CBOE download script (scripts/cache_cboe_spx_0dte.py or similar) may have incorrectly mapped columns from the Athena query results, or the Athena table structure changed.
